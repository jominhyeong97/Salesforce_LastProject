진입점 2개, 처리는 1곳 — LWC 입력바의 ReAct LLM 라우터(AgentRouter→Action→AgentSynthesize) · Agentforce 외부 챗봇 — 모두 동일 6개 Apex Invocable 호출. 코드 중복 0.
Router 진입 — Apex 5줄
// AgentRouterController.cls
String tool = LlmRouter.choose(question);
   // 1단계 — 도구 선택
Map<String,Object> data =
   invokeAction(tool, q);
   // 2단계 — Apex Action 실행
return LlmRouter.synthesize(question, data);
   // 3단계 — 한국어 응답 합성

LLM 응답이 코드 펜스에 갇혔다

PROBLEM · 문제

Apex 단위 테스트는 33/33 통과. 실제 호출 시 JSON 파싱 실패 — Unexpected character '`' code 96.

CAUSE · 원인

LLM이 응답을 ```json ... ``` 마크다운 코드 펜스로 감쌌다. "마크다운 절대 금지" 지시도 무시. 단위 테스트의 mock JSON은 펜스가 없어 못 잡았다.

FIX · 해결

모든 LLM 응답 wrapper에 stripCodeFences 헬퍼 통과 의무화. 정규식으로 백틱·json 키워드 제거 후 JSON.deserialize.

LEARNING · 학습

Apex 테스트 통과가 실제 LLM 통과를 보장하지 않는다. mock 응답과 실제 응답이 다를 때를 항상 가정. 통합 시연이 진짜 테스트.

CAUSE · 원인

컴파일 통과 ≠ 런타임 통과. 표준 필드도 조직마다 활성 여부가 다르다. Schema describe로 검증 후 결정.
